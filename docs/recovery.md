# Recovery

## First Response

Stop automation before changing state:

```sh
sudo systemctl stop edge-agent
sudo nft list ruleset >"/var/tmp/nft-before-recovery-$(date +%Y%m%d%H%M%S).nft"
sudo cp /var/lib/edge-router/state.sqlite "/var/tmp/edge-state-$(date +%Y%m%d%H%M%S).sqlite"
```

If traffic is broken after an apply, remove only the EdgeRoute table:

```sh
sudo nft delete table ip edge_nat
```

That table name is generated by the current renderer. Do not flush the whole ruleset unless you intend to remove unrelated firewall state.

## Disable One Mapping

```sh
edge --db /var/lib/edge-router/state.sqlite map list
edge --db /var/lib/edge-router/state.sqlite map disable <mapping_id>
edge --db /var/lib/edge-router/state.sqlite apply --dry-run
sudo edge --db /var/lib/edge-router/state.sqlite apply --check
sudo edge --db /var/lib/edge-router/state.sqlite apply
```

If check fails, do not apply. Keep the agent stopped and fix the mapping or kernel/nft issue first.

## Restore Database

```sh
sudo systemctl stop edge-agent
sudo cp /var/tmp/edge-state-YYYYMMDDHHMMSS.sqlite /var/lib/edge-router/state.sqlite
sudo chown root:root /var/lib/edge-router/state.sqlite
sudo edge --db /var/lib/edge-router/state.sqlite apply --check
sudo systemctl start edge-agent
```

## Release OCI Addresses

Release OCI resources only after local nft rollback is complete and callers have been moved away from the public IP:

```sh
edge oracle ip release --public-ip-id <public_ip_ocid> --private-ip-id <private_ip_ocid>
```

If database state is lost, recover OCIDs from `edge map list`, OCI Console, or:

```sh
edge oracle ip list --compartment-id <compartment_ocid>
```

## Tailscale Route Recovery

On the home subnet router:

```sh
sudo tailscale up --advertise-routes=192.168.20.0/24
tailscale status
```

Approve the subnet route in the admin console if it was disabled. On the edge host:

```sh
tailscale status
edge tailscale routes
edge tailscale check 192.168.20.42 --ping
ip route get 192.168.20.42
```

The route should use `tailscale0`. If it does not, fix Tailscale route approval/acceptance before touching nftables.

## Kernel Or nft Failure

Symptoms: `nft -c` reports unsupported syntax, missing table support, or invalid argument on a minimal ruleset.

Actions:

```sh
./scripts/nft-min-check.sh
uname -a
nft --version
zgrep CONFIG_NF_TABLES /proc/config.gz 2>/dev/null || true
```

Use a Linux kernel with `CONFIG_NF_TABLES=y` or `m`; install the distro nftables package; load required modules if the distro ships them as modules. macOS and Apple Container cannot validate this path because they do not expose Linux netfilter/nftables to the container.

## Return To Service

```sh
sudo edge --db /var/lib/edge-router/state.sqlite apply --check
sudo edge --db /var/lib/edge-router/state.sqlite apply
sudo systemctl start edge-agent
sudo systemctl status edge-agent --no-pager
```

Keep the pre-recovery nft and SQLite snapshots until traffic has been verified from an external client.
